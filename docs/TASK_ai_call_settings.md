# Задание: вкладка «Настройки ИИ-звонков»

## Цель
Добавить в портал HR PRO AI (D:\HrAiPro, Node/Express + SPA `public/js/app.js`) отдельную вкладку **«ИИ-звонки»** с настройками поведения голосового ИИ-агента (Vapi), и научить систему звонков этим настройкам подчиняться: расписание, повторные звонки, длительность, имя агента, миссия компании.

Настройки — **на уровень вакансии** (`vacancy.process.aiCall`), но с наследованием: см. «Модель наследования».

## Где разместить (ВАЖНО: не в «Интеграциях»)
Во вкладке **«Процесс найма»** вакансии, **рядом с тумблером «Автоворонка»** — добавить **кнопку-шестерёнку «Настройки ИИ-звонков»**. Клик → **модалка** с формой (8 полей ниже) + кнопка «Применить ко всем вакансиям». Модалка — в стиле портала (как модалка «Контекст вакансии» в AI-расшифровках: тёмная, непрозрачная, класс-паттерн `.db-modal`, или существующие модалки app.js).

Место в коде: вкладка процесса рендерится в app.js рядом со строкой `sw('auto', !!proc.auto)` (блок «Автоворонка», см. `body.innerHTML` в функции настроек процесса). Кнопку-шестерёнку ставить в тот же ряд, что и тумблер авто.

## Модель данных — `vacancy.process.aiCall`
```js
{
  agentName: '',            // имя агента; пусто → дефолт по языку (Ева/Ewa/Eva)
  companyMission: '',       // миссия/продукт компании (1–3 предложения); пусто → вопрос о миссии НЕ задаётся
  hoursFrom: '10:00',       // начало окна звонков (локальное время)
  hoursTo: '19:00',         // конец окна звонков
  days: [1,2,3,4,5],        // дни недели, когда звоним (1=Пн … 7=Вс)
  maxDurationMin: 10,       // макс. длительность разговора, мин → обрыв
  retryAfterMin: 60,        // если не подняли трубку — перезвонить через N минут
  retryCount: 3,            // сколько раз пытаться на одном этапе (после N неответов — этап пропускаем)
  offHoursCallback: 'defer' // 'call' — звонить в нерабочее, если сам попросил; 'defer' — перенести на след. рабочий день
}
```

## Модель наследования (обязательно)
- **Глобальный дефолт = «первое указанное значение».** Как только настройки ИИ-звонков заполнены хотя бы у ОДНОЙ вакансии, эти значения по умолчанию применяются ко ВСЕМ вакансиям, у которых своих настроек нет. Реализовать резолвер `resolveAiCall(user, vacancy)`:
  1. если у вакансии есть свой `process.aiCall` (был сохранён вручную) → берём его;
  2. иначе → берём `aiCall` первой (по `createdAt`/`order`) вакансии пользователя, у которой он задан (это и есть «первое указанное значение»);
  3. иначе (ни у кого не задано) → **стандартные дефолты**: рабочие дни Пн–Пт `[1,2,3,4,5]`, время `10:00–19:00`, `maxDurationMin:10`, `retryAfterMin:60`, `retryCount:3`, `offHoursCallback:'defer'`, `agentName:''` (→ Ева), `companyMission:''`. **Форму в модалке при первом открытии предзаполнить этими стандартными значениями** (чтобы поля не были пустыми).
- **Ручное изменение — только для выбранной вакансии.** Сохранение из модалки (`PUT /api/vacancies/:id/process { aiCall }`) ставит флаг «своё» у этой вакансии (напр. хранить сам факт наличия `process.aiCall` = переопределение). Другие вакансии не трогаются.
- **Кнопка «Применить ко всем вакансиям»** в модалке → `POST /api/vacancies/ai-call/apply-all { aiCall }` — копирует текущие значения из модалки в `process.aiCall` ВСЕХ вакансий пользователя (перезаписывает их индивидуальные). Показать подтверждение.

Везде, где раньше брали настройки звонков (`aiCallFor`, референс-роут, планировщик), использовать `resolveAiCall(owner, vac)` вместо чтения одного поля.

## UI-поля вкладки (каждое с подсказкой, i18n ru/pl/en)
1. **Имя агента** — text input, placeholder «Ева». Подсказка: «Как агент представляется в звонке. Пусто — по языку: Ева / Ewa / Eva.»
2. **Миссия компании** — textarea. Подсказка: «1–3 предложения о цели и продукте компании. Если пусто — агент не задаёт вопрос о миссии.»
3. **Время звонков** — два поля времени «с» и «до» (`<input type="time">`). Валидация from < to.
4. **Дни звонков** — 7 переключателей/чипов Пн–Вс (мультивыбор).
5. **Максимальная длительность разговора** — number (мин), 1–30, дефолт 10.
6. **Перезвонить, если не подняли, через** — number (мин), 5–1440, дефолт 60.
7. **Сколько попыток дозвона на этапе** — number, 1–10, дефолт 3. Подсказка: «После стольких неответов звонки по этому этапу прекращаются; на следующем этапе счётчик начинается заново.»
8. **Если попросили перезвонить в нерабочее время** — select/тумблер: «Звонить в нерабочее» / «Перенести на следующий рабочий день» (дефолт — перенести).

Кнопка «Сохранить» → `PUT /api/settings { aiCall: {...} }`.

Пробросить из `aiCallFor`/референс-роута из `settings.aiCall.maxDurationMin`.

### 3. Планировщик (новый модуль `src/call-scheduler.js`)
Заменить прямой `integ.startCall` в `aiCallFor` и в ИИ-референсах на **постановку задачи в очередь**:
- Новая коллекция в БД `db().callQueue = []` (см. `src/db.js` DEFAULT — добавить массив). Элемент:
  ```js
  { id, userId, participantId, kind, refIndex?, callId?, status: 'pending'|'calling'|'done'|'failed'|'stopped',
    attempts: 0, nextAt: ISO, contact?: {...}, createdAt, lastReason? }
  ```
- Функция `enqueueCall({userId, participantId, kind, refIndex?})` — ставит задачу с `nextAt = now` (или ближайшее рабочее окно). `aiCallFor` и `POST /references/ai-call` вызывают её вместо прямого звонка.
- **Тик планировщика** — `setInterval` в server.js (напр. каждые 30–60 сек), функция `tickCallQueue()`:
  1. Для каждой `pending`-задачи, у которой `nextAt <= now`:
     - Проверить **рабочее окно** (`settings.aiCall.hoursFrom/hoursTo/days`, локальное время вакансии/аккаунта). Если сейчас не рабочее время/день → перенести `nextAt` на ближайшее начало окна, `continue`.
     - Разместить звонок `integ.startCall(...)` (промт по kind, maxDuration), сохранить `callId`, `status='calling'`, `attempts++`.
  2. Для `calling`-задач: `integ.getCall(callId)`:
     - завершён с ответом (разговор состоялся) → `status='done'` (для референсов — распарсить результат, как уже сделано в `GET /references/ai-call/:refIndex`).
     - **не подняли трубку** (`endedReason` ∈ `customer-did-not-answer` / `no-answer` / `voicemail` / `customer-busy`): если `attempts < retryCount` → `status='pending'`, `nextAt = now + retryAfterMin`; иначе `status='stopped'` (этап пропущен, больше не звоним по нему).
- **Следующий этап — цикл заново**: счётчик `attempts` привязан к `(participantId, kind)`; на новом `kind` создаётся новая задача с `attempts=0`.
- **offHoursCallback**: если из разговора/структурированного результата ясно, что просили перезвонить в конкретное время (например, поле `convenient_to_talk`/`factual_notes` или отдельный вопрос) вне окна: при `offHoursCallback==='call'` — ставить `nextAt` на запрошенное время как есть; при `'defer'` — переносить на ближайшее рабочее окно.

### 4. Рабочее время — вспомогалки
- `isWorkingTime(settings, date)` и `nextWorkingSlot(settings, from)` — по `hoursFrom/hoursTo/days`. Часовой пояс: пока брать серверный/аккаунта (в TODO — по стране вакансии).

## Критерии приёмки
- Кнопка-шестерёнка стоит рядом с тумблером «Автоворонка» во вкладке «Процесс найма»; открывает модалку с 8 полями, предзаполненными (первое заданное значение или стандартные дефолты).
- Сохранение из модалки меняет настройки ТОЛЬКО этой вакансии; значения переживают перезагрузку.
- Наследование: заполнил у одной вакансии → у других (без своих настроек) по умолчанию берётся это значение. Проверить `resolveAiCall`.
- Кнопка «Применить ко всем вакансиям» перезаписывает настройки всех вакансий пользователя текущими из модалки.
- Ни у кого не задано → используются стандартные рабочие дни (Пн–Пт) и время (10:00–19:00).
- Пустая миссия → в промте afterResult нет вопроса о миссии (проверить сборкой промта).
- Пустое имя агента → в промте/первой реплике Ева/Ewa/Eva по языку; заданное имя — используется.
- Звонок вне окна времени/дня → откладывается до ближайшего рабочего слота (не звонит сразу).
- Неответ → перезвон через `retryAfterMin`, не более `retryCount` раз; после исчерпания — этап помечается `stopped`, звонки по нему не идут; следующий этап начинает счётчик заново.
- `maxDurationMin` уходит в Vapi (`maxDurationSeconds`).
- Синтаксис-чек всех правок; форма и логика планировщика проверены (юнит-тест `isWorkingTime`/`nextWorkingSlot` и сборки промта без миссии; реальные звонки Vapi — вживую не гонять).
